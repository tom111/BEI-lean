# Mathlib PR Plan: Variable Ideals in MvPolynomial

## Overview

Contribute `MvPolynomial.isPrime_span_X_image` and `MvPolynomial.height_span_X_image`
to Mathlib as two small PRs. These prove that the ideal generated by a set of variables
in a multivariate polynomial ring over a field is prime and has height equal to the
cardinality of the variable set.

## Before writing code

1. Join Zulip at `https://leanprover.zulipchat.com/`
2. Post in `#new members` and `#mathlib` describing the planned PR
3. Fork `leanprover-community/mathlib4` on GitHub
4. Clone, set up remotes, run `lake exe cache get`

## PR strategy: split into 2 small PRs

### PR 1: `MvPolynomial.isPrime_span_X_image`
- Statement: The ideal `span(X '' s)` is prime in `MvPolynomial sigma K`
- ~100 lines, no new definitions, extends existing MvPolynomial theory
- File: `Mathlib/RingTheory/MvPolynomial/Ideal.lean` (extend existing) or new file
- PR title: `feat(RingTheory/MvPolynomial): prove variable ideals are prime`

### PR 2 (depends on PR 1): `MvPolynomial.height_span_X_image`
- Statement: `height(span(X '' s)) = s.card`
- Uses primality from PR 1 + Krull's height theorem + chain construction
- PR title: `feat(RingTheory/MvPolynomial): height of variable ideal equals cardinality`

## Style requirements

- PR title format: `feat(RingTheory/MvPolynomial): description in imperative`
- Line length: max 100 characters
- Naming: `snake_case` for theorems (e.g. `isPrime_span_X_image`)
- Docstrings: required on every `def` and public theorem
- Module docstring: required with `## Main results`, `## Tags` sections
- Copyright header: author name, Apache 2.0
- `by` at end of previous line (`:= by`), never standalone
- No `$` (use `<|`), no `sorry`, no empty lines inside declarations
- American English, `Type*` not `Type _`

## Naming conventions

| Theorem | Name |
|---------|------|
| The ideal is prime | `MvPolynomial.isPrime_span_X_image` |
| Height equals card | `MvPolynomial.height_span_X_image` |
| Height upper bound | `MvPolynomial.height_span_X_le` |
| Height lower bound | `MvPolynomial.height_span_X_ge` |
