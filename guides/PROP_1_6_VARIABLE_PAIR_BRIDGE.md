## Exact next goal

Bridge the BEI-specific ideal

- `bipartiteEdgeMonomialIdeal`

to the new generic edge-ideal API

- `MvPolynomial.variablePairIdeal`.

The ideal in `BEI/CohenMacaulay.lean` is already generated by monomials

- `X (Sum.inl i) * X (Sum.inr j)`

with the adjacency/ordering condition that encodes the Herzog–Hibi bipartite graph.

So the likely right next theorem is:

1. define the corresponding set of pairs `edgesGamma G : Set ((BinomialEdgeVars (Fin n)) × (BinomialEdgeVars (Fin n)))`, or a smaller equivalent edge-set wrapper;
2. prove
   `bipartiteEdgeMonomialIdeal G = MvPolynomial.variablePairIdeal edgesGamma`;
3. if that is clean, immediately derive the minimal-prime characterization for
   `bipartiteEdgeMonomialIdeal G` by transport from
   `MvPolynomial.minimalPrime_variablePairIdeal_iff`.


## Why this is the right next packet

This is smaller and cleaner than trying to finish Proposition 1.6 in one round.

It converts the new squarefree infrastructure into something directly usable by the
BEI file. After this bridge lands, the remaining CM branch becomes a clearer
graph-combinatorial/equidimensionality problem:

- show all minimal vertex covers of the HH bipartite graph have equal size;
- convert that to equidimensionality / Cohen–Macaulayness for the quotient;
- then finish the initial-ideal transfer.


## Recommended implementation plan

### Step 1: define the bipartite edge set explicitly

Read the definition:

- `bipartiteEdgeMonomialIdeal` in `BEI/CohenMacaulay.lean`

and define the matching edge set in the exact same variables.

The edge set should match the generators literally, not only up to a later rewrite.
That will make the bridge theorem a short unfolding / `span` argument.

Prefer the smallest direct definition. Do not introduce a large new graph structure
unless the later HH combinatorics genuinely needs it.

### Step 2: prove ideal equality by unfolding generators

The bridge theorem should ideally be nearly definitional:

- unfold `bipartiteEdgeMonomialIdeal`
- unfold `MvPolynomial.variablePairIdeal`
- prove the generator sets coincide up to straightforward existential repackaging

If you need a helper lemma, keep it small and local.

### Step 3: transport the minimal-prime theorem

Once the ideal equality is available, prove a theorem of the form:

- minimal primes of `bipartiteEdgeMonomialIdeal G`
  are exactly `span (X '' S)` for minimal vertex covers of the associated pair set

Do not over-design the statement. Package it in the form that will be easiest to use
for the next Proposition 1.6 step.

## False routes / cautions

  landing the direct bridge.

  unfolding and extensionality.
