# Guide: Section 4 CI-Ideals and Robustness

## Scope

Section 4 is not one theorem. It is a chain of definitions and reductions ending in
several corollaries.

Claude should not start Section 4 by trying to formalize probability theory. The paper's
actual algebraic content is much narrower:

- define the CI-polynomials for the special binary-output setup;
- identify them with a binomial edge ideal of a graph on the state space;
- import Section 2 and Section 3 results;
- restate the conclusions in statistical language.


## The paper's formalizable core

The truly formal algebraic statements are:

1. for binary output `d_0 = 2`, each relevant CI polynomial is a `2 x 2` minor;
2. a robustness specification determines a graph on `X_[N]`;
3. the CI ideal of that specification equals the binomial edge ideal of the graph;
4. therefore the CI ideal is radical;
5. its minimal primes are indexed by the same `S` satisfying Corollary 3.9.

The later "statistical interpretation" paragraphs should be treated as downstream prose
unless a precise formal notion is introduced.

## Formalization stages

## Stage 1: finite state spaces and index types

Model the state space for the input variables as a finite product.

For Lean ergonomics, do not start with arbitrary dependent products unless necessary.
Use a concrete representation that makes enumeration and equality easy.

Possible options:

1. tuples over `Fin`;
2. a finite sigma/product encoding of assignments;
3. a direct finite type parameter `Ω` representing `X_[N]`.

Recommendation:

- start with an abstract finite type `Ω` for the state space of the non-output variables;
- define the graph directly on `Ω`;
- only later add a bridge saying `Ω` can be instantiated as the product state space.

This keeps the algebra from drowning in indexing details.


## Stage 2: the binary-output polynomial ring

For binary output, the polynomial ring has one variable for each pair:

- output bit `b : Fin 2`
- state `ω : Ω`

This is exactly the same shape as the BEI polynomial ring:

- one `x_ω`
- one `y_ω`

So define the dictionary:

- `x ω` corresponds to `p_{1,ω}`
- `y ω` corresponds to `p_{2,ω}`

This should be a ring equivalence or at least a notation-level identification.


## Stage 3: CI statement for a single partition

For one partition `S ∪ T = [N]`, the paper associates a graph whose edges connect
states agreeing on the `T`-coordinates.

Formalize:

1. a relation `AgreeOn T ω ω'`;
2. the graph generated by that relation;
3. the CI ideal for the statement `X_0 ⟂ X_S [X_T]`;
4. a theorem that this ideal equals the binomial edge ideal of the graph.

This theorem is the key bridge.


## Stage 4: robustness specifications

A robustness specification is a finite family of partitions `(S_i, T_i)`.

Formalize:

1. the graph attached to each statement;
2. the union graph attached to the whole specification;
3. the CI ideal of the family as the ideal generated by the single-statement ideals;
4. equality of that ideal with the BEI of the union graph.


## Stage 5: import the BEI results

Once the ideal equality is in place:

- radicality follows from `corollary_2_2`;
- prime decomposition follows from `theorem_3_2`;
- minimal prime characterization follows from `corollary_3_9`.

This stage should be easy if the bridge theorems are set up well.


## Stage 6: statistical restatements

Only after the algebraic bridge exists should Claude attempt the prose-level corollaries:

- structural zeros;
- minimal primes correspond to sets `S`;
- connected-component interpretation.

These statements require careful choices about what is actually formalized:

- ideals and varieties only;
- or also real nonnegative probability distributions.

Recommendation:

- formalize the algebraic side only;
- keep the statistical interpretation in docstrings unless a serious probability layer
  is wanted.

Preferably done first:

## Common traps

- Do not formalize general probability theory unless the paper actually uses it.
- Do not start with full-blown random variables; start with polynomial coordinates.
- Do not hard-code the full product-state-space machinery before the BEI bridge works.


## Definition of done

Section 4 should count as formalized when:

1. the CI ideal of a robustness specification with binary output is identified with a
   BEI;
2. radicality and prime decomposition corollaries are transferred cleanly;
3. the statistical interpretation is separated clearly from the algebraic theorem layer.
