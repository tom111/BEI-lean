## Landed definition

Work with a monomial-ideal predicate on ideals of `MvPolynomial`.

Suggested local definition:

```lean
namespace Ideal

def IsMonomial {σ R : Type*} [CommSemiring R] (I : Ideal (MvPolynomial σ R)) : Prop :=
  ∀ p ∈ I, ∀ d ∈ p.support, MvPolynomial.monomial d (1 : R) ∈ I

end Ideal
```

Reason:

- it matches the existing mathlib membership lemmas;
- it avoids introducing a separate monomial-ideal structure too early;
- it is exactly what the BEI project will want when reasoning about initial ideals and
  other monomial ideals inside `MvPolynomial`.


## Prime monomial ideals: theorem target

### Paper/math statement

A monomial ideal in a polynomial ring over a domain is prime if and only if it is
generated by a set of variables.

### Recommended Lean statement

Because the generating set of variables need not be finite, the right endpoint should
use a `Set σ`, not a `Finset σ`.

Suggested statement:

```lean
namespace Ideal

theorem IsMonomial.isPrime_iff_eq_span_X_image
    {σ R : Type*} [CommRing R] [IsDomain R]
    {I : Ideal (MvPolynomial σ R)} (hI : I.IsMonomial) :
    I.IsPrime ↔ ∃ s : Set σ, I = Ideal.span (MvPolynomial.X '' s) := by
  ...

end Ideal
```

Equivalent repackaging if preferred:

```lean
theorem IsMonomial.isPrime_iff
    ...
    I.IsPrime ↔ ∃ s : Set σ, I = Ideal.span (MvPolynomial.X '' s)
```

This theorem was the right first `toMathlib` endpoint, and it is now proved in
`toMathlib/MonomialIdeal.lean`.


## Prime case: implementation plan

This section is now historical; the plan below is what was carried out to land the
prime classification. It remains useful context for the primary case.

### Step 1: generalize `isPrime_span_X_image` from `Finset` to `Set`

Current local theorem:

- `MvPolynomial.isPrime_span_X_image (s : Finset σ)`

Recommended replacement or companion theorem:

```lean
theorem MvPolynomial.isPrime_span_X_image_set
    {σ R : Type*} [CommRing R] [IsDomain R] (s : Set σ) :
    (Ideal.span (MvPolynomial.X '' s : Set (MvPolynomial σ R))).IsPrime
```

The current `killCompl` proof should generalize cleanly by replacing the complement
finset with the subtype `{i // i ∉ s}`.

### Step 2: from primeness, define the variable set

For a prime monomial ideal `I`, define:

```lean
let s : Set σ := { i | MvPolynomial.X i ∈ I }
```

Then show:

```lean
I = Ideal.span (MvPolynomial.X '' s)
```

### Step 3: key monomial lemma

The crucial lemma is:

```lean
lemma exists_variable_mem_of_monomial_mem_prime
    {I : Ideal (MvPolynomial σ R)} (hI : I.IsPrime)
    {d : σ →₀ ℕ} (hd : d ≠ 0)
    (hmem : MvPolynomial.monomial d (1 : R) ∈ I) :
    ∃ i, d i ≠ 0 ∧ MvPolynomial.X i ∈ I
```

This should be proved by induction on `Finsupp.degree d`, factoring
`monomial d 1 = X i * monomial d' 1` for some `i` in the support.

### Step 4: prove the classification

Use `hI : I.IsMonomial` plus the key lemma and
`MvPolynomial.mem_ideal_span_X_image`:

- if `p ∈ I`, then every support monomial of `p` lies in `I`;
- hence each support monomial has some variable factor lying in `s`;
- therefore `p ∈ Ideal.span (X '' s)`.

The reverse inclusion is immediate from the definition of `s`.


## Primary monomial ideals: mathematical characterization

This is now the live next target.

Yes, there is a clean characterization, but it is a bit more delicate to package
cleanly in Lean than the prime case.

The right conceptual picture is:

1. the radical of a monomial primary ideal is a monomial prime ideal, hence generated by
   variables;
2. once `radical I = Ideal.span (X '' s)`, the variables outside `s` act as
   non-zero-divisors on the monomial basis of the quotient;
3. equivalently, for monomials outside `I`, multiplying by a variable outside `s` keeps
   you outside `I`.


## Primary case: recommended theorem packaging

Do **not** try to jump straight to full primary decomposition.
First prove a characterization of primary monomial ideals.

### Suggested first theorem

```lean
namespace Ideal

theorem IsMonomial.isPrimary_iff
    {σ R : Type*} [CommRing R] [IsDomain R]
    {I : Ideal (MvPolynomial σ R)} (hI : I.IsMonomial) :
    I.IsPrimary ↔
      ∃ s : Set σ,
        I.radical = Ideal.span (MvPolynomial.X '' s) ∧
        ∀ d : σ →₀ ℕ, MvPolynomial.monomial d (1 : R) ∉ I →
          ∀ i, i ∉ s →
            MvPolynomial.monomial (d + Finsupp.single i 1) (1 : R) ∉ I

end Ideal
```

Interpretation:

- `s` is the set of variables in the radical;
- outside those variables, multiplication by a variable preserves non-membership for
  monomials.

This is the monomial-ideal version of “all zero divisors are nilpotent”.


## Why this is the right primary statement

It separates the theorem into two accessible parts:

1. `I.radical` is a variable ideal;
2. primary-ness is the outside-the-radical non-zero-divisor condition on monomials.

That is much more implementable than trying to formalize a broad textbook sentence about
all associated primes at once.


## Primary case: implementation plan

Status:
- Phase 1 is still open.
- Phase 2 is now done as `Ideal.isPrimary_monomial_criterion`.
- Phase 3 remains open and is blocked on the same leading-term machinery as Phase 1.

### Phase 1: prove radicals of prime monomial ideals are variable ideals

This becomes immediate once the prime classification theorem above is in place.

### Phase 2: prove `I.IsPrimary ->` monomial criterion

This direction should be easy.

Assume:

- `hprim : I.IsPrimary`
- `I.radical = Ideal.span (X '' s)`
- `monomial d 1 ∉ I`
- `i ∉ s`

If `monomial (d + single i 1) 1 ∈ I`, then

```text
X i * monomial d 1 ∈ I
```

and primary-ness would force `X i ∈ I.radical`, contradiction.

Status: done in `toMathlib/MonomialIdeal.lean` as
`Ideal.isPrimary_monomial_criterion`.

### Phase 3: prove the converse

This is the only genuinely nontrivial part.

Recommended route:

1. assume the monomial criterion above;
2. to prove `I.IsPrimary`, start from `a * b ∈ I` and `a ∉ I`;
3. use `hI : IsMonomial` and a monomial order / leading-monomial argument to reduce to a
   support monomial of `a` outside `I` and a support monomial of `b` whose product must
   lie in `I`;
4. conclude that some variable appearing in that support monomial of `b` lies in `s`;
5. hence that support monomial lies in `Ideal.span (X '' s) = I.radical`;
6. use `MvPolynomial.mem_ideal_span_X_image` to conclude `b ∈ I.radical`.

The reason to use a leading-monomial argument is to avoid coefficient-cancellation issues
in the support of `a * b`.

Current blocker:

- The needed leading-term statement also seems to be what blocks
  `Ideal.IsMonomial.radical_isMonomial`.
- Concretely, the missing algebra is about coefficients in powers:
  for a lex-maximal support monomial `d_max` of `p`, one wants to isolate the
  coefficient of `n • d_max` in `p^n` and show it is `(p.coeff d_max)^n`.
- That appears to require infrastructure around `coeff_mul`, `Finset.antidiagonal`,
  and induction on powers, rather than a short BEI-specific shortcut.

Recommended next packet:

1. isolate and prove the leading-term coefficient lemma for powers in `MvPolynomial`;
2. use it to prove `Ideal.IsMonomial.radical_isMonomial`;
3. return to the converse direction of the primary iff.

## Warnings

- Do not state the prime classification with `Finset σ` unless you also assume enough
  finiteness on the set of variables occurring in the ideal. The correct general target
  uses `Set σ`.
- Do not try to jump directly to full primary decomposition.
- Do not hide coefficient-cancellation issues in the converse direction of the primary
  theorem; this is exactly where a monomial-order / leading-term argument is useful.
- Do not confuse “radical is generated by variables” with “the ideal itself is generated
  by variables”; the former is the primary case, the latter is the prime case.
